# MSPM0 双电机通信协议

## 1. 串口参数

```text
波特率：500,000 baud
数据位：8
校验位：无
停止位：1
流控：无
格式：8N1
```

TI 板的 RX 和 TX 均配置为 500 Kbps。

## 2. 通用帧格式

```text
SOF0  SOF1  SEQ  CMD  LEN  PAYLOAD...  CRC_LO  CRC_HI
A5    5A
```

| 字段 | 长度 | 含义 |
|---|---:|---|
| `SOF0` | 1字节 | 固定为 `0xA5` |
| `SOF1` | 1字节 | 固定为 `0x5A` |
| `SEQ` | 1字节 | 帧序号，可循环递增 |
| `CMD` | 1字节 | 命令字 |
| `LEN` | 1字节 | Payload长度 |
| `PAYLOAD` | N字节 | 有效数据 |
| `CRC_LO` | 1字节 | CRC16低字节 |
| `CRC_HI` | 1字节 | CRC16高字节 |

### 数据约定

- CRC算法：Modbus CRC16；
- CRC计算范围：从 `SOF0` 开始，到 Payload 最后一个字节结束；
- 多字节整数：Payload 内采用**大端序**，高字节在前；
- CRC发送顺序：低字节在前，高字节在后。

# 3. 达妙 H723 → MSPM0

## 3.1 设置两个电机PWM

```text
CMD = 0x10
LEN = 4
```

Payload：

```text
pwm_a_i16_be  pwm_b_i16_be
```

即：

| Payload字节 | 含义 |
|---|---|
| 0～1 | A电机PWM，`int16_t`，大端 |
| 2～3 | B电机PWM，`int16_t`，大端 |

PWM范围：

```text
-4000 ~ +4000
```

含义：

- 正数：一个方向；
- 负数：相反方向；
- 绝对值决定PWM占空比；
- `4000` 对应100%占空比；
- `2000` 对应50%占空比；
- `200` 对应5%占空比。

完整设置PWM帧长度：

```text
2字节帧头 + 1字节SEQ + 1字节CMD + 1字节LEN
+ 4字节Payload + 2字节CRC = 11字节
```

### 超时保护

PWM命令必须持续刷新。

若连续 100 ms 没有收到有效的 PWM 或 STOP 命令：

```text
A电机PWM = 0
B电机PWM = 0
```

两个电机将被强制停止。

## 3.2 停止命令

```text
CMD = 0x11
LEN = 0
```

收到后立即令 A、B 两个电机停止。

## 3.3 Ping测试命令

```text
CMD = 0x12
LEN = 0
```

MSPM0收到正确的 Ping 帧后，回复：

```text
CMD = 0x91
LEN = 0
```

Ping只用于验证串口收发和协议解析，不解除电机命令超时状态。

# 4. MSPM0 → 达妙 H723

## 4.1 双电机反馈

```text
CMD = 0x90
LEN = 13
```

Payload格式：

```text
字节0～3    A电机累计编码值
字节4～7    B电机累计编码值
字节8～9    A电机速度
字节10～11  B电机速度
字节12      状态字节
```

| 字段 | 类型 | 字节数 | 字节序 | 单位/含义 |
|---|---|---:|---|---|
| `encoder_count_a` | `int32_t` | 4 | 大端 | A电机上电后的累计编码计数 |
| `encoder_count_b` | `int32_t` | 4 | 大端 | B电机上电后的累计编码计数 |
| `speed_a_cps` | `int16_t` | 2 | 大端 | A电机编码计数/秒 |
| `speed_b_cps` | `int16_t` | 2 | 大端 | B电机编码计数/秒 |
| `status` | `uint8_t` | 1 | — | 当前运行状态 |

### 速度计算

- 速度数据每 1 ms 更新一次；
- 使用最近 10 ms 的编码增量计算速度；
- 速度单位为 `counts/s`，即每秒编码计数；
- 暂时没有直接换算为 RPM。

### 状态字节

| 位 | 含义 |
|---|---|
| bit0 | 已收到有效电机命令，且尚未超时 |
| bit1 | 电机命令超时，A/B PWM已强制归零 |
| bit2～bit7 | 保留 |

常见状态值：

```text
0x01：通信命令有效
0x02：通信超时，电机已停止
```

### 反馈频率与带宽

反馈周期：

```text
1 ms，即1 kHz
```

完整反馈帧长度：

```text
2 + 1 + 1 + 1 + 13 + 2 = 20字节
```

500Kbps、8N1下，每字节在线路上占10位：

```text
20字节 × 10位 × 1000帧/秒 = 200,000 bit/s
```

占TX线路带宽的40%。

## 4.2 Ping回复

```text
CMD = 0x91
LEN = 0
```

表示MSPM0已经正确收到并解析Ping命令。
